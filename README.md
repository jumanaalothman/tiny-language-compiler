# TINY Language Compiler

## Overview

This repository contains a compiler construction project for the TINY programming language. The project includes lexical analysis and syntax analysis phases implemented using Flex and Bison.

The goal of the project is to recognize TINY source programs, tokenize the input, and parse the program according to the defined grammar rules.

## Project Phases

### Phase 1: Lexical Analyzer

Phase 1 implements a lexical analyzer for the TINY language using Flex.

The lexical analyzer:

- Recognizes reserved words
- Identifies valid identifiers and numbers
- Recognizes operators and symbols
- Ignores whitespace and comments
- Prints tokens in order
- Detects lexical errors such as unrecognized symbols and invalid identifiers

### Phase 2: Syntax Analyzer / Parser

Phase 2 extends the project by adding a parser using Bison.

The parser:

- Uses tokens generated by the lexical analyzer
- Defines grammar rules for the TINY language
- Parses program declarations and statements
- Supports assignment, if, while, read, and write statements
- Supports arithmetic and logical expressions
- Reports syntax errors when the input does not follow the grammar

## Supported TINY Language Constructs

- Program structure

- Array type
- Assignment statements
- If-else statements
- While loops
- Read statements
- Write statements
- Arithmetic expressions
- Boolean conditions
- Logical operators

## How to Build and Run Phase 1

```bash
flex tiny.l
gcc lex.yy.c -lfl -o tiny
./tiny < test.tiny.txt
```

## How to Build and Run Phase 2

```bash
bison -d tiny.y
flex tiny.l
gcc y.tab.c lex.yy.c -lfl -o tiny_parser
./tiny_parser < test.tiny.txt
```

If your system generates `tiny.tab.c` and `tiny.tab.h` instead of `y.tab.c` and `y.tab.h`, use:

```bash
bison -d -o y.tab.c tiny.y
flex tiny.l
gcc y.tab.c lex.yy.c -lfl -o tiny_parser
./tiny_parser < test.tiny.txt
```
